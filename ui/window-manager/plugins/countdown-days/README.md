# Countdown Days（日历计时）

`countdown-days` 是基于 ChillPatcher `window-manager` 的日历事件插件。  
你可以在日历上选日期、添加事件，并用卡片查看“还剩 X 天 / 已经过去 X 天 / 就在今天”。

## 功能概览

- 日历交互：按月切换、点击“今天”、点击年月弹出月份选择器。
- 事件管理：快速添加、编辑、删除、置顶（PIN/UNP）。
- 智能天数：根据系统当天日期自动计算倒计时与正计时。
- 视图模式：完整模式（日历 + 事件 + 分页）、收起模式（仅事件区）、缩小模式（前 `1/2/4` 张卡片）。
- 外观设置：预设主题 + 自定义颜色。
- 卡片信息密度：`简洁`（标题+天数）/ `标准`（日期+操作按钮+标题+天数）。
- 事件存储路径可配置，支持自动迁移到新路径。

## 目录位置

将本目录放到 ChillPatcher 仓库中：

- `ui/window-manager/plugins/countdown-days/`

## 安装与构建

在 `ui/window-manager` 下执行：

```bash
npm install
npm run build
```

开发时推荐：

```bash
npm run watch
```

## 使用说明

### 1) 添加事件

1. 在左侧日历点击某一天。
2. 在“快速添加事件”输入标题。
3. 点击“添加到选中日期”。

事件会显示在右侧“全部事件（置顶优先）”列表中。

### 2) 卡片操作

- 点击卡片本体：定位到该事件日期。
- `PIN/UNP`：置顶或取消置顶。
- `EDT`：打开编辑弹窗（可改标题和日期）。
- `DEL`：进入删除确认覆盖层，点击 `YES/NO` 确认。

### 3) 分页

列表底部使用 `<< < 1 2 3 > >>` 翻页。  
每页条目数会随可用高度自动计算，并受“卡片信息密度”影响：

- `简洁`：单卡更矮，通常每页显示更多。
- `标准`：信息更全，单卡更高。

### 4) 视图模式

- 顶部左侧“收起”：切换到中间模式（只看卡片列表）。
- 中间模式顶部“展开”：恢复完整模式。
- 顶部“小卡 1/2/4”：设置缩小模式显示卡片数量。

## 设置项说明

点击顶部“设置”进入设置面板。

### 主设置

- 事件 JSON 路径：事件数据文件路径（支持相对路径）。
- 颜色：进入“自定义颜色”子菜单。
- 卡片信息密度：`简洁` / `标准`。
- 分页设置：每页事件数（1-20，自动分页上限）与一页标签数（页码按钮数量，3-9）。

### 自定义颜色

- 预设主题：深海蓝 / 夜森林 / 落日红 / 钢铁灰 / 日间亮色 / 摩卡棕 / 薄荷绿 / 樱花粉 / 米白纸 / 极夜黑。
- 自定义颜色项：整体背景、主文字颜色、天数高亮色。

## 数据文件与存储逻辑

默认文件：

- 配置文件：`window-states/countdown-days.json`
- 事件文件：`window-states/countdown-days-events.json`

默认是“配置与事件分离存储”：

- 配置文件保存 UI 配置。
- 事件文件保存事件数组。

如果你把“事件 JSON 路径”设置为 `window-states/countdown-days.json`，会进入“合并存储”模式。  
当你修改事件路径时，插件会自动将当前事件写入新路径（迁移）；旧文件不会自动删除。

事件文件结构示例：

```json
{
  "events": [
    {
      "id": "evt-abc123",
      "title": "考试",
      "targetDate": "2026-04-30",
      "type": "countdown",
      "createdAt": "2026-03-22T09:30:00.000Z",
      "pinned": true
    }
  ]
}
```

## 常见问题

### Q1: 今天日期不对

插件按宿主环境系统时间计算“今天”。  
请先确认系统日期/时区是否正确，再重开插件窗口。

### Q2: 改了设置但界面没变化

先点击“应用设置”或“应用颜色”。  
若仍未更新，建议重开 `window-manager` 或重新编译前端输出。

### Q3: 为什么我在简洁模式看不到编辑/删除按钮

这是设计行为。  
简洁模式只显示“标题 + 天数”，用于高密度浏览。  
切回标准模式可使用 `PIN/EDT/DEL` 操作。
